# UI Change Proposal

Before editing any CSS or HTML layout:

1. Describe the current state of the element being changed
2. Propose 2-3 alternative approaches with tradeoffs for each:
   - Complexity (simple vs clever)
   - Mobile behavior (stacked/wrapped preferred over scroll/carousel)
   - Visual intuitiveness
3. Wait for the user to pick an approach before editing any file
4. After approval, implement only the chosen approach — no extras
